# 概率对战 Demo

一个基于概率分布采样的双人对战小游戏（玩家 vs 电脑），已实现为 `tkinter` 可视化程序。

## 核心规则

- 双方各自选择一个概率分布模型与参数。
- 每局各采样一次，并计算判定值：
  - `判定值 = 采样值 - 理论期望`
- 判定值更大者获胜。

这意味着双方判定值都满足 `E[判定值] = 0`，胜负只由分布形状与波动结构决定。

## 已实现分布（玩家可选）

- 均匀分布 `Uniform(a, -a)`（只选 `a<=0`）
- 正态分布 `Normal(0, sigma)`（只选 `sigma>0`）
- 指数分布 `Exp(lambda)`（自动中心化）
- 泊松分布 `Poisson(lam)`（自动中心化）
- 二项分布 `Binomial(n, p)`（自动中心化）
- 伽马分布 `Gamma(k, theta)`（自动中心化）
- 贝塔分布 `Beta(alpha, beta)`（自动中心化）

## 电脑对手

- 均值工程师（偏稳健）
- 爆发投机者（高波动）
- 离散操盘手（离散流派）
- 泊松计数师（计数流派）
- 全随机模拟器（随机分布 + 随机参数）

## 策略分析（玩家最关心）

### 是否存在“最佳策略”？

- 不存在一个对所有电脑都同时最优的单一策略。
- 但对每个固定电脑对手，都存在稳定的“近似最优应对参数”。

原因：本游戏比较的是中心化后的判定值（`采样值-理论期望`），期望已被消除，胜负主要由分布形状、离散程度和尾部结构决定。

### 基于当前版本的实测推荐

以下结果来自蒙特卡洛模拟（每组 `150,000` 局，固定随机种子），适用于本仓库当前代码版本中的电脑参数配置。

| 电脑对手 | 推荐策略（玩家） | 估计胜率 |
| --- | --- | --- |
| 均值工程师 | `Binomial(n=32, p=0.978076422412949)` | `~53.0%` |
| 爆发投机者 | `Normal(0, sigma=0.08477435925090383)` | `~62.6%` |
| 离散操盘手 | `Binomial(n=1, p=0.3)` | `~57.7%` |
| 泊松计数师 | `Binomial(n=17, p=0.046748029747690545)` | `~59.1%` |

### 懒人通用策略

如果你不想按敌人切换参数，可用：

- `Binomial(n=32, p=0.978076422412949)`

这组参数在前三个固定对手上都不错，但对“泊松计数师”明显变弱（约 `52.4%`），因此仍建议按对手切换。

### 对全随机模拟器的建议

- 先锁定敌人后，界面会显示本轮敌人的分布和参数。
- 你再按上面的思路“针对当局配置”选参数，通常强于固定一套参数硬打。

## 可视化流程（树形回退）

操作顺序为：

1. 锁定敌人
2. 锁定模型
3. 锁定参数
4. 开始判定（支持重复判定）

每局后可以：

- 再次判定（保持敌人、模型、参数不变）
- 返回参数选择（保持敌人、模型不变）
- 返回模型选择（保持敌人不变）
- 返回敌人选择（重选敌人，模型和参数重置）

## 运行方式

### 依赖

- Python 3.10+（推荐）
- 标准库 `tkinter`（Windows 通常默认可用）

### 启动

```bash
python probability_battle_demo.py
```
